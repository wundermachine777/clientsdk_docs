Introduction

The gamecore is a rest endpoint providing game data. With the ClientSDK developers are capable of using the gamecore REST service without the need to make manual http requests

Initialization

new ClientSDK(false, {baseUrl: '/'})

and

clientSDK.init(providerToken, callback)

should be called once, while the game is loading. It does not interact with the DOM and can be called at an early stage.

The baseUrl parameter can be modified if the games are uploaded to a shared environment. In the default configuration, clientSDK tries to communicate with the gamecore at $HOST/op/

For LPCL (Luckypoint client library) based games providerToken is provided by lpcl.getSessionID(); Please be aware of the additional events required by the lpcl (like lpcl.reportGameReady()) [see https://github.com/luckypointlabs/luckypoint-client-library]

Playing

Base Game

Basically a the game is initialized with init() and keeps spin()-ning until something special happens.

Once the user wins, the frontend has to issue a capture() request at the point when the win should be realized. (When reels are done spinning. A win animation is played, ...)

If a win is not captured, it gets sent automatically with the next spin() request by the ClientSDK.

If the user quits, or the network is disconnected before the capture request has successfully finished, the last win will be paid out to the user upon next init()

Bonus Game

The bonus game works like the basegame, but the wins are paid out at the end of the bonus game, not at the end of the spin. (Freespins count as one bonus game and are paid out at the last spin/summary screen) The bonus game is active while the parameter end is false, and spinsLeft is above 0.

Events (Smashing)

Once the user issues a smash, a triggerEvent(64, {smashReel: 0, smashRow: 0}) shoud be called.

  • smashReel contains the 0-indexed position of the reel (left to right)
  • smashRow contains the 0-indexed position of the row (top to bottom)

For an example see the GameCore Server-Side Documentation
